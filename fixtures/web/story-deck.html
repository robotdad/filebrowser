FileBrowser

A web-based file browser & editor — FastAPI backend, Preact SPA front-end, no build step required.

FastAPI Preact No Build Step Multi-format Preview Integrated Terminal
Architecture

Built to be simple.
Designed to be fast.

A single-process server that serves both the API and the SPA — no webpack, no npm build, no containers required to develop.

⚡

FastAPI Backend

  • Async Python, uvicorn ASGI server
  • Streams large files without buffering
  • PAM authentication (bypass-able via nginx header injection in test environments)
  • Pluggable auth middleware
🧩

Preact SPA — No Build Step

  • HTM tagged templates render JSX at runtime
  • ES modules loaded directly from the browser
  • No webpack, Vite, or transpilation pipeline
  • Edit a .js file, reload, see changes
🔀

Unified Entry Point

One uvicorn main:app command. Static assets, API routes, and WebSocket terminal all served from the same process on a single port.

🐧

Linux Native

PAM login, POSIX paths, real terminal via pty + WebSocket. Runs anywhere Python 3.11+ and a POSIX shell exist.

File Support

Eight format families.
One interface.

The preview engine selects a renderer based on MIME type. Every format gets a dedicated experience.

.md Markdown — live preview, WYSIWYG (Tiptap), raw source 3 tabs
code Source files — syntax-highlighted via Prism.js (50+ languages) highlight
.png .jpg Images — inline preview, zoom, pan native
.mp4 .mp3 Audio & Video — HTML5 player, no transcoding native
.pdf PDF — embedded viewer via browser PDF engine native
.dot Graphviz DOT — interactive render via d3-graphviz + WASM WASM
.html HTML — served as text/plain (source view only) known gap
terminal Pseudo-terminal — full shell via WebSocket + xterm.js WS
Key Features

Power where
you need it.

📝

Dual Markdown Editor

Three tabs per file: Preview (rendered HTML), WYSIWYG (Tiptap rich text), Source (raw text). Switch without losing content.

⚠ WYSIWYG round-trips drop fenced code blocks & GFM tables — known issue under active testing.

🕸

Interactive Graphviz

DOT files render live in the browser. No server-side graphviz binary required — d3-graphviz compiles the layout engine to WebAssembly and runs it client-side.

⌨️

Command Palette Ctrl+K

Fuzzy search across all files in the browse root. Open, navigate, and switch files without touching the mouse. Keyboard-first design throughout.

🖥

Integrated Terminal

Full xterm.js terminal connected to a real PTY via WebSocket. Session cookie bridge required — auth token must be pre-seeded before the WS handshake.
